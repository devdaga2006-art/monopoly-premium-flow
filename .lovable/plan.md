# Migration: TanStack Start → Vite SPA for Netlify

## Goal
Keep every page, component, style, and asset exactly as it looks today, but remove TanStack Start / Router / Cloudflare Workers so the project builds to a plain static SPA that Netlify can host with zero config beyond the existing `netlify.toml`.

## What changes

### 1. Dependencies
- **Remove**: `@tanstack/react-router`, `@tanstack/react-start`, `@tanstack/router-devtools`, `@tanstack/router-plugin`, `@cloudflare/vite-plugin`, `@lovable.dev/vite-tanstack-config`, `wrangler`, `react-helmet-async` (if present).
- **Add**: `react-router-dom`, `react-helmet-async`, plain `vite`, `@vitejs/plugin-react`, `@tailwindcss/vite` (Tailwind v4 keeps working with `src/styles.css`).

### 2. Build tooling
- New `vite.config.ts` using plain `defineConfig` with `react()`, `tailwindcss()`, `tsconfigPaths()`, and the `@` → `src` alias. Output goes to `dist/client` so the existing `netlify.toml` keeps working.
- New `index.html` at project root as the SPA entry, with `<div id="root">` and `<script type="module" src="/src/main.tsx">`.
- New `src/main.tsx` that mounts `<App />` inside `BrowserRouter`, `HelmetProvider`, and `QueryClientProvider`.
- Delete `wrangler.jsonc`, `src/server.ts`, `src/start.ts`, `src/router.tsx`, `src/routeTree.gen.ts`, `src/lib/error-page.ts`, `src/lib/error-capture.ts` (if Worker-specific).
- Remove `npm run dev`'s Cloudflare bits; scripts become `vite`, `vite build`, `vite preview`.

### 3. Routing
- New `src/App.tsx` with `<Routes>` mapping every existing path:
  - `/`, `/about`, `/contact`, `/faq`, `/industries`, `/infrastructure`, `/products`, `/products/:slug`, `/locations/:slug`, `/why-us`, plus a 404.
- The sitemap moves from a server route to a build-time generated static file at `public/sitemap.xml` (generated by a small `scripts/generate-sitemap.mjs` invoked from the `build` script). Same URLs as today.

### 4. Per-page refactor (mechanical, UI unchanged)
For each file in `src/routes/*`:
- Replace `createFileRoute(...)` export with a default-exported React component.
- Replace `Link` import from `@tanstack/react-router` with `Link` from `react-router-dom` (`to` prop is identical).
- Replace `useParams()` from TanStack with the `react-router-dom` equivalent.
- Replace the route's `head()` meta with `<Helmet>` tags rendered at the top of the component (titles, descriptions, OG, Twitter, JSON-LD via `<script type="application/ld+json">`).
- Move files from `src/routes/` to `src/pages/` to avoid confusion with the old router convention.

### 5. Root layout
- `src/App.tsx` wraps `<Routes>` with the existing `Header`, `Footer`, `WhatsAppFab`, `CallFab`, `Toaster` — same JSX as today's `RootComponent`.
- Global `<Helmet>` in `App.tsx` carries the Organization/WebSite JSON-LD currently in `__root.tsx` so structured data is preserved on every page.

### 6. Netlify
- `netlify.toml` already publishes `dist/client` and rewrites `/*` → `/index.html`. No changes needed.
- `.env` is still not required — no env vars are used in source.

## What stays exactly the same
- Every component under `src/components/**`, every style in `src/styles.css`, every asset, all data files (`src/data/*`), the design system, the hero, the MONOPOLYMERS branding, all SEO copy, JSON-LD content, breadcrumb helpers, location/product detail pages.

## Deploy steps after merge
1. Push to GitHub.
2. On Netlify: "Add new site → Import from Git", build command `npm run build`, publish dir `dist/client` (already in `netlify.toml`).
3. Done — SPA with client-side routing and pre-rendered `index.html`.

## Risk / tradeoffs
- This is a **SPA, not SSR**. Initial HTML is the same shell for every URL; meta tags are injected client-side via Helmet. Google renders JS so SEO still works, but if pure server-rendered HTML per route is a hard requirement, we'd instead need a Netlify-compatible SSR adapter (bigger lift, more moving parts). Given the request ("remove tanstack, keep UI"), SPA is the right tradeoff.
- The sitemap is generated at build time from the same `locations`/`products` data, so it stays accurate.
